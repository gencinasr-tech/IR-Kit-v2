# Contributing | Contribuyendo

## EN

Thank you for your interest in contributing to IR-Kit-v2!

### How to Contribute

1. **Fork** this repository
2. Create a **feature branch**: `git checkout -b feature/your-feature-name`
3. **Commit** your changes: `git commit -m 'Add: description of your changes'`
4. **Push** to your fork: `git push origin feature/your-feature-name`
5. Open a **Pull Request** with a clear description

### Contribution Guidelines

- **Documentation**: Keep it clear and bilingual (EN/ES) where possible
- **Case Studies**: Sanitize all real data and remove sensitive information
- **Code/Scripts**: Comment your code and test before submitting
- **Issues**: Search for duplicates before opening a new one

### Types of Contributions We Welcome

- ✅ Improvements to the IR execution protocol
- ✅ New case studies (with anonymized data)
- ✅ Tool guides and troubleshooting notes
- ✅ Additional playbooks and checklists
- ✅ Typo fixes and documentation improvements
- ✅ Translations or localization

### Code of Conduct

Be respectful, professional, and constructive. This is a security-focused community.

## ES

¡Gracias por tu interés en contribuir a IR-Kit-v2!

### Cómo Contribuir

1. **Fork** este repositorio
2. Crea una **rama de feature**: `git checkout -b feature/tu-nombre-feature`
3. **Commit** tus cambios: `git commit -m 'Add: descripción de tus cambios'`
4. **Push** a tu fork: `git push origin feature/tu-nombre-feature`
5. Abre un **Pull Request** con una descripción clara

### Directrices de Contribución

- **Documentación**: Mantenerla clara y bilingüe (EN/ES) si es posible
- **Casos de Estudio**: Anonimiza todos los datos reales y elimina información sensible
- **Código/Scripts**: Comenta tu código y prueba antes de enviar
- **Issues**: Busca duplicados antes de abrir uno nuevo

### Tipos de Contribuciones que Buscamos

- ✅ Mejoras al protocolo de ejecución de respuesta ante incidentes
- ✅ Nuevos casos de estudio (con datos anonimizados)
- ✅ Guías de herramientas y notas de troubleshooting
- ✅ Playbooks y checklists adicionales
- ✅ Corrección de typos y mejoras de documentación
- ✅ Traducciones o localización

### Código de Conducta

Sé respetuoso, profesional y constructivo. Esta es una comunidad enfocada en seguridad.
